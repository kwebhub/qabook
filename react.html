Состояние и хуки

  • Состояние (state) необходимо для изменения значения внутри компонента и сохранения этого значения между рендорами.
  • Сохранять и изменять можно только состояния (state).
  • Конкретный state контролируется и доступен только конкретному компоненту.
  • Состояние определяется пользователем и должно быть простым объектом JS.
  • Состояние можно изменять только через setState(), присвоить состоянию значение напрямую можно только в конструкторе.
  • Компонент может передать своё состояние вниз по дереву в виде пропсов дочерних компонентов.
  • Этот процесс называется 'нисходящим' ('top-down') или 'однонаправленным' ('unidirectional') потоком данных.
  • Если несколько компонентов должны отражать одни и те же изменяющиеся данные, то нужно поднимать общее состояние до ближайшего общего предка.
  • Чтобы собрать данные из нескольких дочерних элементов, или чтобы дать возможность двум компонентам общаться, нужно объявить общее состояние внутри родительского компонента.
  • Родительский компонент может передать состояние обратно дочерним элементам с помощью пропсов.
  • Это поддерживает синхронизацию дочерних компонентов друг с другом и с родительским компонентом.
  • Хуки - это функции, с которыми можно 'подцепиться' к состоянию и к методам жизненного цикла React из функциональных компонентов.
  • Хуки не работают в классах, а используются вместо них.
  • Хуки вызывают только на верхнем уровне (не в циклах, условиях и т.д.)
  • Хуки вызывают только из функциональных компонентов React, не из обычных JS-функций.
  • Если имя функции начинается с use (useSomething) и использует другие хуки, то такая функция это пользовательский хук.

Хук эффекта выполняет побочные действия:

  • загрузка данных
  • оформление подписки и изменение DOM вручную
  • представляет собой совокупность методов componentDidMount, componentDidUpdate и componentWillUnmount
  • Эффекты позволяют выполнить дополнительный код после того, как React обновил DOM, т.е. после каждого рендера.
  • Любой эффект может возвратить функцию, которая сбросит его.
  • React будет сбрасывать эффект перед тем, как компонент размонтируется.

Например оформление и отмена подписки:

import React , {useState, useEffect} from 'react'
function FriendStatus(props) {
  const [isOnline, setIsOnline] = useState(null)
  useEffect(() => {
    function handleStatusChange(status) {
      setIsOnline(status.isOnline)
    }
    ChatAPI.subscribeToFriendStatus(
      props.friend.id,
      handleStatusChange
    )

    // сброс эффекта
    return function cleanup() {
      ChatAPI.unsubscribeFromFriendStatus(
        props.friend.id,
        handleStatusChange
      )
    }
  })
  if (isOnline === null) {
    return 'Загрузка…'
  }
  return isOnline ? 'В сети' : 'Не в сети'
}
  • Чтобы React выполнял эффект только в случае изменения нужен второй аргумент в виде массива, с которым будет сравниваться значение первого аргумента.
  • При каждом рендере react сравнивает новое значение переменной в первом аргументе со старым значением переменной, указанным во втором аргументе (в массиве).
// стрелочная функция это первый аргумент эффекта
useEffect(() => {
  function handleStatusChange(status) {
    setIsOnline(status.isOnline)
  }

  // это второй аргумент эффекта со старым значением переменной, по которому будет сравнение первого аргумента
  ChatAPI.subscribeToFriendStatus(

    // повторно подписаться только если props.friend.id изменился
    props.friend.id,
    handleStatusChange
  )
  return() => {
    ChatAPI.unsubscribeFromFriendStatus(
      props.friend.id,
      handleStatusChange

}

Управляемый компонент

  • Если форму обрабатывать с помощью JS-функции, у которой есть доступ к введённым данным, то такая форма называется 'управляемым компонентом'.
  • Библиотека Formik.
  • Или, другими словами, управляемый компонент - это элемент формы рендер, поведение при пользовательском вводе и значение которого контролирует React.
class NameForm extends React.Component{
  constructor(props){
    super(props)
    this.state={value:' '}
    this.handleChange=this.handleChange.bind(this)
    this.handleSubmit=this.handleSubmit.bind(this)
  }
  handleChange(event) {
    this.setState({value: event.target.value})
  }
  handleSubmit(event){
    alert('Отправленное имя: ' + this.state.value)
    event.preventDefault()           // отменяем действие по умолчанию
  }
  render() {
    return (
      < form onSubmit={this.handleSubmit}>
        < label>
          Имя:
          < input type="text" value={this.state.value} onChange={this.handleChange} />
        < /label>
        < input type="Submit" value="Отправить" />
      < /form>
    )
  }
}

в атрибуте value всегда будет отражаться React состояние this.state.value, а каждое нажатие клавиш через handleChange будет это состояние изменять и React будет обновлять поле ввода по мере печатанья.

  • useReducer - альтернатива для useState, управляет состоянием через функцию-редюсер, которую пишешь сам.

  • Это событие обычно используется для отслеживания изменений в полях ввода и выполнения определенных действий при этих изменениях.

  • useContext в React - это хук, который позволяет компонентам получать доступ к значению контекста, предоставляемому родительским компонентом через <Context.Provider>.

  text: string;
}

const initState: StateType = { count: 0, text: '' }

const enum REDUCER_ACTION_TYPE {
  INCREMENT,
  DECREMENT,
  NEW_INPUT,
}

type ReducerAction = {
  type: REDUCER_ACTION_TYPE,
  payload?: string,
}

const reducer = (state: StateType, action: ReducerAction): StateType => {
  switch (action.type) {
    case REDUCER_ACTION_TYPE.INCREMENT:
      return { ...state, count: state.count + 1 }
    case REDUCER_ACTION_TYPE.DECREMENT:
      return { ...state, count: state.count - 1 }
    case REDUCER_ACTION_TYPE.NEW_INPUT:
      return { ...state, text: action.payload ?? '' }
    default:
      throw new Error()

}

const useCounterContext = (initState: StateType) => {
  const [state, dispatch] = useReducer(reducer, initState)

  const increment = useCallback(() => dispatch({ type: REDUCER_ACTION_TYPE.INCREMENT }), [])

  const decrement = useCallback(() => dispatch({ type: REDUCER_ACTION_TYPE.DECREMENT }), [])

  const handleTextInput = useCallback((e: ChangeEvent< HTMLInputElement>) => {
    dispatch({
      type: REDUCER_ACTION_TYPE.NEW_INPUT,
      payload: e.target.value
    })
  }, [])

  return { state, increment, decrement, handleTextInput }
}

const initContextState: UseCounterContextType = {
  state: initState,
  increment: () => { },
  decrement: () => { },

function App() {
  const hours = new Date().getHours()
  let timeOfDay

  if (hours < 12) {
    timeOfDay = "morning"
  } else if (hours >= 12 && hours < 17) {
    timeOfDay = "afternoon"
  } else if (hours < 21) {
    timeOfDay = "evening"
  } else {
    timeOfDay = "night"
  }
}

return (
  < h1>Good {timeOfDay}!< /h1>

import React from "react";
import padsData from "./pads";
import Pad from "./Pad";

export default function App() {

  // начальное состояние - это массив из файла pads.js
  const [pads, setPads] = React.useState(padsData);

  // toggle the on property of the pad with id of clicked pad
  function toggle(id) {
    setPads((prevPads) => prevPads.map((item) => {
        return item.id === id ? { ...item, on: !item.on } : item;
      })
    );
  }

  const buttonElements = pads.map((pad) => (
    < Pad
      toggle={toggle}
      id={pad.id}
      key={pad.id}
      color={pad.color}
      on={pad.on}
    />
  ));

  return (
    < main>
      < div className="pad-container">{buttonElements}< /div>
    < /main>
  );
}

// Pad.jsx
export default function Pad(props) {
  return (
    < button

      // инлайн стили в react
      style={{ backgroundColor: props.color }}

      // если props.on = true, то к кнопке добавляется класс on, который устанавливает opacity: 1;
      className={props.on ? "on" : undefined}

      // функция toggle передана через props из родительского компонента
      onClick={() => props.toggle(props.id)}

      // если в onClick поместить просто props.toggle(props.id), то toggle будет срабатывать без клика по кнопке
    >< /button>
  );
}

// WindowTracker.jsx
import React from "react";

export default function WindowTracker() {
  const [windowWidth, setWindowWidth] = React.useState(window.innerWidth);

  React.useEffect(() => {
    function watchWindowWidth() {
      setWindowWidth(window.innerWidth);
    }

    // т.к. слушатель установлен на окно, он слушает события даже при отключенном компоненте
    window.addEventListener("resize", watchWindowWidth);

    return () => {

      // при отключении компонента useEffect вернёт функцию, которая удалит слушатель с окна
      window.removeEventListener("resize", watchWindowWidth);
    };
  }, []);

  return < h1>Window width: {windowWidth}< /h1>;
}

// App.jsx
// установить пакеты npm install --save nanoid react-confetti
import { useState, useRef, useEffect } from "react";
import Die from "./Die";
import { nanoid } from "nanoid";
import Confetti from "react-confetti";

export default function App() {
  /*
  * сохранить в состояние dice массив из 10 объектов, который вернула функция generateAllNewDice
  * т.к. каждый клик по компоненту Die вызывает рендер, то будет срабатывать useState и менять начальное состояние
  * useState(() => generateAllNewDice()) - позволяет не менять начальное состояние при каждом рендере, а только при вызове generateAllNewDice
  * вызов generateAllNewDice происходит при вызове rollDice когда gameWon = true
  * useRef чтобы при первом рендере обозначить элемент DOM элемент (button), для обращения к нему в useEffect
  */
  const [dice, setDice] = useState(() => generateAllNewDice());
  const buttonRef = useRef(null);

  /*
  * встроенный метод массива every проверяет чему равно свойство isHeld у каждого объекта die
  * если свойства isHeld каждого объекта die имеют значение true, то сравнить значения свойства value каждого объекта die
  * если значения свойства value (цифра) каждого объекта die равно значению свойста value первого объекта die из массива состояния dice, то вернуть true
  * если значения свойств isHeld каждого объекта die равны true и результат сравнения значений свойств value (цифра) тоже true,
  * то присвоить переменной gameWon значение true
  */
  const gameWon =
    dice.every((die) => die.isHeld) &&
    dice.every((die) => die.value === dice[0].value);

  useEffect(() => {
    /*
    * сработает только при зменении значения gameWon
    * если значение gameWon = true, то переместить фокус на кнопку "New Game" для тех кто использует клавиатуру без мышки
    * useRef возвращает объект buttonRef, у которого свойство current содержит нужный нам DOM элемент (button)
    */
    if (gameWon) {
      buttonRef.current.focus();
    }
  }, [gameWon]);

  function generateAllNewDice() {
    /*
    * вернуть массив из 10 цифр равных 0, перебрать каждый элемент массива и присвоить объект
    * в объекте свойство value - это массив произвольно генерированных цифр от 1 до 6
    * свойство объекта isHeld - нажата кнопка die или нет
    * nanoid генерирует уникальный id
    */
    return new Array(10).fill(0).map(() => ({
      value: Math.ceil(Math.random() * 6),
      isHeld: false,
      id: nanoid(),
    }));
  }

  function rollDice() {

    /*
    * если gameWon = false, то к каждому значению массива из состояния dice применить тернарный оператор
    * если свойство isHeld объекта die из массива состояния dice = true, то оставить в объект die без изменений
    * если свойство isHeld объекта die из массива состояния dice =false, то изменить его свойство value на произвольную цифру от 1 до 6
    * setDice при gameWon = false вызывает функцию map внутри себя, map возвращает объект, который setDice сохраняет в массив состояния die
    * если gameWon = true, то вызвать функцию generateAllNewDice, которая перезаполнит массив состояния dice новыми объектами
    * setDice(generateAllNewDice() - меняет начальное состояние die только если gameWon = true, т.е. когда выиграл и нужно начать новую игру
    */
    if (!gameWon) {
      setDice((oldDice) => oldDice.map((die) =>
          die.isHeld ? die : { ...die, value: Math.ceil(Math.random() * 6) }
        )
      );
    } else {
      setDice(generateAllNewDice());
    }
  }

  function hold(id) {
    /*
    * срабатывает при нажатии на компонент Die
    * получает id нажатого компонента Die
    * меняет свойство isHeld нажатого компонента на противоположное значение true/false
    */
    setDice((oldDice) => oldDice.map((die) =>
        die.id === id ? { ...die, isHeld: !die.isHeld } : die
      )
    );
  }

  /*
  * сохранить компоненты Die в массив
  * перебрать объекты массива состояния dice и присвоить значения свойств этих объектов пропсам компонентов Die
  * при переборе массива компонентов Die, у каждого компонента должен быть уникальный key
  * пропс hold для каждого компонента Die получает id из каждого объекта массива состояния dice
  */
  const diceElements = dice.map((dieObj) => (
    < Die
      key={dieObj.id}
      value={dieObj.value}
      isHeld={dieObj.isHeld}

      // х.з. как оно работает
      hold={() => hold(dieObj.id)}
    />
  ));

  return (
    < main>

      // если gameWon = true, то рендерить компонент Confetti из установленного пакета
      {gameWon && < Confetti />}
      < div aria-live="polite" className="sr-only">
        {gameWon && (

          // для скринридера
          < p>Congratulations! You won! Press "New Game" to start again.< /p>
        )}
      < /div>
      < h1 className="title">Tenzies< /h1>
      < p className="instructions">
        Roll until all dice are the same. Click each die to freeze it at its
        current value between rolls.
      < /p>

      // вывести массив компонентов Die
      < div className="dice-container">{diceElements}< /div>

      /* 
      * ref={buttonRef} - useRef помечает DOM элемент (button), к которому мы обратимся в useEffect
      * при срабатывании useEffect переместить сюда фокус, при нажатии вызвать функцию rollDice
      *
      */
      < button ref={buttonRef} className="roll-dice" onClick={rollDice}>

        // если gameWon = true, то изменить надпись на кнопке
        {gameWon ? "New Game" : "Roll"}
      < /button>
    < /main>
  );
}

// App.jsx
/*
* {languages} - {} нужны для деструктиризации массива из файла languages.js, т.к. экспорт из файла без default
* установить пакеты npm install --save clsx react-confetti
*/
import { useState } from "react";
import { clsx } from "clsx";
import { languages } from "./languages";
import { getFarewellText, getRandomWord } from "./utils";
import Confetti from "react-confetti";

export default function AssemblyEndgame() {
  // State values
  /**
  * начальное состояние задаётся только привызове getRandomWord
  * автоматически при рендере не задаётся
  * useState(() => getRandomWord()) - lazy state initialization срабатывает только при первом рендере
  */
  const [currentWord, setCurrentWord] = useState(() => getRandomWord());
  const [guessedLetters, setGuessedLetters] = useState([]);

  // Derived values
  // количество попыток, чтобы последний язык остался незатемнённым
  const numGuessesLeft = languages.length - 1;

  // создать счётчик ошибок из длины массива состояния guessedLetters без букв содержащихся в загадоном слове
  const wrongGuessCount = guessedLetters.filter((letter) => !currentWord.includes(letter)).length;

  // сделать массив из букв загаданного слова и сравнить есть ли каждая из них в массиве нажатых букв, true/false
  const isGameWon = currentWord.split("").every((letter) => guessedLetters.includes(letter));

  // если счётчик ошибок больше или равен числу возможных попыток, то isGameLost = true
  const isGameLost = wrongGuessCount >= numGuessesLeft;

  // игра окончена (true) если выиграл или проиграл
  const isGameOver = isGameWon || isGameLost;

  // получить букву по последнему индексу массива нажатых букв, т.е. получить последнюю нажатую букву
  const lastGuessedLetter = guessedLetters[guessedLetters.length - 1];

  /*
  * содержит ли загаданное слово последнюю нажатую букву
  * т.е. проверить нажатую букву при каждом нажатии
  * ! чтобы привести к true/false
  */
  const isLastGuessIncorrect = lastGuessedLetter && !currentWord.includes(lastGuessedLetter);

  // Static values
  /*
  * буквы алфавита для клавиатуры
  */
  const alphabet = "abcdefghijklmnopqrstuvwxyz";

  /*
  * добавит нажатую букву в массив состояния guessedLetters если такой ещё нет
  */
  function addGuessedLetter(letter) {
    setGuessedLetters((prevLetters) => prevLetters.includes(letter) ? prevLetters : [...prevLetters, letter]);
  }

  /**
  * срабатывает при нажатии на кнопку "New Game"
  * выбирает произвольное слово и обнуляет массив состояния нажатых букв
  */
  function startNewGame() {
    setCurrentWord(getRandomWord());
    setGuessedLetters([]);
  }

  /*
  * перебираю массив из файла languages.js
  * класс lost будет добавляться к lang если его index меньше счетчика ошибок
  * т.е. затемнено будет столько языков сколько ошибок в счётчике
  * знак меньше, т.к. index начинается с 0
  */
  const languageElements = languages.map((lang, index) => {
    const isLanguageLost = index <  wrongGuessCount;
    const styles = {
      backgroundColor: lang.backgroundColor,
      color: lang.color,
    };

    // добавить класс lost при isLanguageLost = true
    const className = clsx("chip", isLanguageLost && "lost");

    /*
    * возвращаю каждый язык как отдельный элемент со своим цветом и именем
    * использую имя языка как ключ для функции map, т.к. в файле languages.js у языков нет свойства id
    */
    return (
      < span className={className} style={styles} key={lang.name}>
        {lang.name}
      < /span>
    );
  });

  /*
  * currentWord.split("") - отделяет каждую букву из слова и возвращает массив букв
  * для ключа функции map использовать индекс массива букв, т.к. порядок букв в слове не меняется, т.е. индекс каждой буквы в массиве не меняется
  */
  const letterElements = currentWord.split("").map((letter, index) => {

    /**
    * если игра проиграна, то покажет все перебираемые буквы загаданного слова
    * если игра не проиграна, то покажет только те перебираемые буквы, которые есть в массиве нажатых букв
    */
    const shouldRevealLetter = isGameLost || guessedLetters.includes(letter);

    /**
    * если игра проиграна окрашивает ненажатые буквы в загаданном слове
    */
    const letterClassName = clsx(isGameLost && !guessedLetters.includes(letter) && "missed-letter");
    return (
      /*
      * если shouldRevealLetter = true, то показать буквы загаданного слова
      */
      < span key={index} className={letterClassName}>
        {shouldRevealLetter ? letter.toUpperCase() : ""}
      < /span>
    );
  });

  /*
  * разделит строку alphabet на массив где каждая буква это отдельный элемент
  * называет каждый элемент массива букв (каждую букву в массиве) letter и передает в функцию map
  * при переборе букв map(letter) проверяю есть ли такая буква в массиве состояния guessedLetters, если есть, то isGuessed = true
  * если перебираемая буква есть в массиве состояния currentWord и isGuessed = true, то isCorrect = true, если нет, то isWrong = true
  * функция clsx из установленного пакета создаёт объект, в котором ключи это css классы и возвращает класс, у которого значение true
  */
  const keyboardElements = alphabet.split("").map((letter) => {
    const isGuessed = guessedLetters.includes(letter);
    const isCorrect = isGuessed && currentWord.includes(letter);
    const isWrong = isGuessed && !currentWord.includes(letter);
    const className = clsx({
      correct: isCorrect,
      wrong: isWrong,
    });

    return (
      < button
        className={className}
        key={letter}

        // если игра окончена, то отключает клавиатуру
        disabled={isGameOver}

        // чтобы таб на нажатые буквы не перемещался
        aria-disabled={guessedLetters.includes(letter)}

        // при табе на букву "А" клавиатуры скринридер скажет "Letter А"
        aria-label={`Letter ${letter}`}

        /*
        * onClick={addGuessedLetter}, то addGuessedLetter получит на вход event со всеми атрибутами
        * onClick={() => addGuessedLetter(letter)} - вызывает addGuessedLetter и передает её на вход только букву, нажали
        */
        onClick={() => addGuessedLetter(letter)}
      >
        {letter.toUpperCase()}
      < /button>
    );
  });

  const gameStatusClass = clsx("game-status", {
    won: isGameWon,
    lost: isGameLost,
    farewell: !isGameOver && isLastGuessIncorrect,
  });

  /**
  * если игра не окончена и нажатая буква неправильная, то получить сообщение для пользователя из файла utils
  * в getFarewellText передать для подстановки в сообщение имя языка с индексом счетчик ошибок - 1
  */
  function renderGameStatus() {
    if (!isGameOver && isLastGuessIncorrect) {
      return (
        < p className="farewell-message">
          {getFarewellText(languages[wrongGuessCount - 1].name)}
        < /p>
      );
    }

    if (isGameWon) {
      return (
        < >
          < h2>You win!< /h2>
          < p>Well done! 🎉< /p>
        < />
      );
    }
    if (isGameLost) {
      return (
        < >
          < h2>Game over!< /h2>
          < p>You lose! Better start learning Assembly 😭< /p>
        < />
      );
    }

    return null;
  }

  return (
    < main>
      {/* отрендерить конфети при выигрыше */}
      {isGameWon && < Confetti recycle={false} numberOfPieces={1000} />}
      < header>
        < h1>Assembly: Endgame< /h1>
        < p>
          Guess the word within 8 attempts to keep the programming world safe
          from Assembly!
        < /p>
      < /header>

      {/*
      * секция присутствует в разметке всегда класс css держит постоянный размер секции даже если она пустая
      * renderGameStatus вставляет при рендере одно из трех сообщений: проиграл, выиграл, null
      * aria-live="polite" role="status" - это ally, т.е. Accessibility - доступность (таб, скринридер и т.д.)
      */}
      < section aria-live="polite" role="status" className={gameStatusClass}>
        {renderGameStatus()}
      < /section>

      < section className="language-chips">{languageElements}< /section>

      < section className="word">{letterElements}< /section>

      {/*
        * Combined visually-hidden aria-live region for status updates
        * для скринридера, на экране не видно
        */}
      < section className="sr-only" aria-live="polite" role="status">
        < p>
          {currentWord.includes(lastGuessedLetter)
            ? `Correct! The letter ${lastGuessedLetter} is in the word.`
            : `Sorry, the letter ${lastGuessedLetter} is not in the word.`}
          You have {numGuessesLeft} attempts left.
        < /p>
        < p>
          Current word:{" "}
          {currentWord
            .split("")
            .map((letter) =>
              guessedLetters.includes(letter) ? letter + "." : "blank."
            )
            .join(" ")}
        < /p>
      < /section>

      < section className="keyboard">{keyboardElements}< /section>

      {isGameOver && (
        < button className="new-game" onClick={startNewGame}>
          New Game
        < /button>
      )}
    < /main>
  );
}

Весь базовый функционал redux умещается в малюсенький файл

function createStore(reducer, initialState) {
  let state = initialState
  return {
    dispatch: action => { state = reducer(state, action) },
    getState: () => state
  }
}
createStore возвращает объект с полем “dispatch”
dispatch это функция, которая применяет редюсер к нашему текущему состоянию: (newState = reducer(state, action)) 
// Инициализация хранилища
function todosReducer(state, action) {
  switch (action.type) {
    case 'ADD_TODO':
      return [
        ...state,
        {
          id: action.id,
          text: action.text,
          completed: false
        }
      ]
    case 'TOGGLE_TODO':
      return state.map(todo => {
        if (todo.id === action.id) {
          return { ...todo, completed: !todo.completed }
        }
        return todo
      })
      default: return state
  }
}

Метод combineReducers позволяет вместо того, чтобы создавать один огромный reducer для всего состояния приложения сразу, разбивать его на отдельные модули.

function counterReducer(state, action) {
  if (action.type === 'ADD') {
    return state + 1
  } else {
    return state
  }
}

const reducer = combineReducers({
  todoState: todoReducer, 
  counterState: counterReducer
})

const initialState = {
  todoState: [],
  counterState: 0,
}

const store = createStore(reducer, initialState)

// Если упростить
function reducer(state, action) {
  return {
    todoState: todoReducer(state, action),
    counterState: counterReducer(state, action),
  }
}

Но этот код плохо масштабируется: если у нас 2 “подсостояния”, нам нужно дважды написать (state, action), а хорошие программисты так не делают. Если редюсоров и стэйтов много, то через обработку коллекции редюсеров:

function combineReducers(reducersMap) {
  return function combinationReducer(state, action) {
    const nextState = {}
    Object.entries(reducersMap).forEach(([key, reducer]) => {
      nextState[key] =  reducer(state[key], action)
    })
    return nextState
  }
}
  • middleware в разрезе redux - это какая-то штука, которая слушает все dispatch и при определенных условиях делает что-то.
  • applyMiddleware переопределяет метод dispatch, добавляя перед (или после) обновлением состояния какую-то пользовательскую логику.
const createStoreWithMiddleware = applyMiddleware(somemiddleware)(createStore)
const store = createStoreWithMiddleware(reducer, initialState)

Самый популярный middleware от создателей redux - redux-thunk, чаще всего используют при асинхронных операциях. Его смысл сводится к тому, что можно передавать в store.dispatch сложные функции.

const thunk = store => dispatch => action => {
  if (typeof action === 'function') {
    return action(store.dispatch, store.getState)
  }
  return dispatch(action)
}
  • метод createStore принимал атрибуты (reducer, initialState) и возвращал объект с ключами { dispatch, getSate }
  • метод applyMiddleware берёт уже реализованный метод createStore и переопределяет его возвращаемое значение:
function applyMiddleware(middleware) {
  return function createStoreWithMiddleware(createStore) {
    return (reducer, state) => {
      const store = createStore(reducer, state)
      return {
        dispatch: action => middleware(store)(store.dispatch)(action),
        getState: store.getState,
      }
    }
  }
}
